# Compass モバイル「今日」画面 設計スペック

## 概要

Compass スマホ版の中核画面。目的は「タスク管理」ではなく「今日を前に進めること」。
Any Planner の時間軸 UI をベースに、Compass 独自のボール（受け渡し）概念を自然に統合する。

参考: https://any-planner.com/

## 設計原則

- 画面の主役は今日のタイムライン
- 情報量は最小限。管理ツールっぽさ、設定画面っぽさを消す
- 白/黒/グレー基調。アクセント色は現在時刻ラインの黄色のみ
- 余白大きめ、角丸大きめ、薄い境界線、影なし
- ボールは独立した表示概念ではなく、カードに対する操作の結果として存在する

## 概念モデル

### タイムラインに出るもの = 自分がアクションを取るもの

タイムラインには「自分が今日やること」だけが並ぶ。
相手待ちのタスクは出ない（自分がアクションを取れないため）。

例外: 催促日が今日の相手待ちタスクは出る（催促する = 自分のアクション）。

### カードに対する操作は2つだけ

| 操作 | ジェスチャー | 意味 | 結果 |
|------|-------------|------|------|
| 完了 | 右端の○タップ | やった | カード消える。done。 |
| 渡す | 右スワイプ | 自分のパートは終わり、次は相手 | 今日から消え、あとで整理トレーに沈む |

- どのカードでも両方の操作ができる
- 「平面図を書く」→ ○タップで完了
- 「田中さんに図面確認する」→ 確認した → 右スワイプで渡す → 自分の今日から消える
- システムが決めるのではなく、ユーザーが文脈で判断する
- どちらも undo 可能（後述のフィードバック仕様参照）

### 左スワイプ（引き取る）

タイムライン上のカードは全て「自分のアクション」なので、左スワイプ（引き取る）は不要。
左スワイプが意味を持つのは**あとで整理トレー内の相手待ちアイテム**のみ:

- トレー内の「→ 田中  図面FB」を左スワイプ → ボールを自分に引き戻す
- 引き戻したアイテムはトレーに残る（タイムラインに配置するかはユーザーが決める）

### 渡す操作の対象（誰に渡すか）

- タスクに `ballHolder` が設定済みで、自分以外の人が記録されている → その人に渡す（確認なし）
- タスクの `assignee`（担当者）が自分以外 → assignee に渡す（確認なし）
- 上記どちらにも該当しない → スワイプ後に小さいシートで渡し先を入力
  - 直近で渡した相手の候補を表示（1タップで選択可能）
  - 新しい名前も入力可能

### 要件定義書との差分

要件定義書（`docs/plans/2026-03-17-mobile-app-requirements.md`）では4タブ構成（今日/ボール/あとで整理/設定）を想定していた。
本設計では以下の理由で3タブ（工程表/今日/設定）に変更:

- ボールは独立画面ではなく、カード操作の結果として統合
- 相手待ちアイテムはあとで整理トレーに同居
- あとで整理はタブではなくボトムシートとして常時アクセス可能

要件定義書の「ボール画面」が担っていた内容の行き先:
- 自分ボール → 今日のタイムラインに表示（アクション対象だから）
- 相手待ち → あとで整理トレー内にフラットに表示
- 期限超過 → あとで整理トレー内で赤テキスト表示
- 今日催促するもの → 今日のタイムラインに表示（催促 = 自分のアクション）

## 画面構成

### レイアウト全体

画面は**1つの連続した流れ**。カード分割やセクション分割はしない。
日付 → 週 → タイムライン → ボトムシートが自然に続く。

```
┌──────────────────────────────┐
│                              │
│  18  3月                     │  ← 大きな日付（画面の主役）
│  2026          < 今日 >      │  ← 年、前日/翌日ナビ
│                              │
│  日  月  火  水  木  金  土   │
│  15  16  17 ⬤18 19  20  21  │  ← 週の日付列
│                              │
│  8:00                        │  ← 時刻ラベル（左、小さくグレー）
│   ┌───────────────────┐  ○  │  ← タスクカード + 完了チェック
│   │ 施主打合せ資料作成   │     │
│   │ 8:00 - 9:00   1h  │     │
│   └───────────────────┘     │
│  9:00                        │
│                              │
│      空き 2時間              │  ← 空き時間ブロック
│      [見積確認] [メール返信]  │  ← チップ提案
│                              │
│  11:00                       │
│   ┌───────────────────┐  ○  │
│   │ 田中さんに図面確認   │     │
│   │ 11:00 - 11:30  30m│     │
│   └───────────────────┘     │
│                              │
│  ═══ 12:21 ═══════════════   │  ← 現在時刻ライン（黄色）
│                              │
│      空き 5時間30分          │
│      [現場写真整理] [書類提出]│
│                              │
├──────────── ▔▔▔ ────────────┤  ← ドラッグハンドル
│  あとで整理          4件     │  ← ボトムシート（常にここに見える）
├──────────────────────────────┤
│  🏠 工程表   ≡ 今日   ⚙ 設定│  ← BottomNav（3タブ）
└──────────────────────────────┘
            (+)                   ← FAB
```

### ヘッダー / 日付エリア

- 「APDW Project Compass」などのアプリ名は**表示しない**
- 日付の数字が画面最大のテキスト（Any Planner の "28" と同様）
- 「今日」ボタン（pill型）、<> で前日/翌日
- 週の日付列: 日〜土、当日は黒丸塗りつぶし
- 日付エリアとタイムラインの間にカード境界やセクションタイトルを**入れない**

### タイムライン

Any Planner 式の連続時間軸。

- 左側に時刻ラベル（8:00, 9:00, ... 小さくグレー）
- タスクカードがその時刻位置に配置される
- タスクがない時間帯は圧縮表示（等間隔ではない）
  - 空き時間ブロックを「空き 2時間」のテキスト + チップで表示（高さは固定、時間に比例しない）
  - チップはあとで整理トレーから「ここに入りそうなもの」を提案
  - チップ選択基準: 所要時間が空き時間に収まるもの → 期限が近い順に最大3件
  - チップタップ → その時間枠に配置
- 現在時刻: 黄色の横断ライン + 時刻バッジ（Any Planner と同様）
- 初回表示時、現在時刻ラインが画面中央付近に来るよう自動スクロール
- 現在時刻ラインは1分ごとに更新

#### 時間配置のロジック

多くのタスクは日付のみ（`予定開始日`）で、時刻データを持たない。
時間配置は以下のロジックで行う:

1. `startTime` を持つタスク → その時刻に配置
2. `startTime` を持たないタスク → 既存の自動配置ロジックを使用
   - 業務開始時刻から順に、`工数見積(h)` 分の枠を確保して積み上げ
   - 既に配置済みのタスクやカレンダーイベントを避ける
   - 間に `MOBILE_TIMELINE_GAP_MINUTES`（20分）のバッファ
3. ユーザーがチップタップやドラッグで手動配置した場合 → `startTime` を書き込み、以降は固定

### タスクカード

- 白背景、角丸大きめ（12-16px）、薄いグレーの border、影なし
- 内容: タスク名（太字）、時間範囲、所要時間
- 右端に完了チェック○（タップで完了）
- 右スワイプで「渡す」操作
- 情報は最小限。PJ名・担当者などは**出さない**（タップして詳細で見る）

### 空き時間ブロック

- 背景色なし、テキストのみで軽く表示
- 「空き 2時間」を中央にグレーテキスト
- その下にチップ（pill型ボタン）で、あとで整理の中から候補を提案
- チップは3-4個まで。多すぎると騒がしい

### 現在時刻ライン

- 黄色の水平線がタイムライン全幅を横断
- 左端に現在時刻バッジ（黄色背景、黒文字）
- Any Planner の表現をそのまま踏襲

## あとで整理ボトムシート

### 基本動作

- **常に画面下部に1行分が見えている**（「あとで整理 4件」のヘッダー行）
- ドラッグハンドル（小さい横棒）で上スワイプして展開
- 展開するとタイムラインの上にオーバーレイ
- BottomNav の上に位置する

### 中身

**フラットなリスト。セクション分けなし。**

- 未整理のタスク（タイトルだけで雑に入れたもの）
- 渡して待っているタスク（相手待ち）
- 両者を同列に並べる

相手待ちのアイテムは、カード内にさりげなく「→ 田中  催促3/20」のように小さく表示するだけ。
それだけで「待ち」と区別できる。

```
├──────────── ▔▔▔ ────────────┤
│ あとで整理              4件  │
│                              │
│  ● 見積書の確認    30m    ○  │
│  ● 田中 → 図面FB  催促3/20  │
│  ● 現場写真整理           ○  │
│  ● 書類提出        15m   ○  │
│                              │
├──────────────────────────────┤
```

### あとで整理からの操作

- タップ → 詳細/編集
- ○タップ → 完了
- 右スワイプ → 渡す
- 長押しまたはドラッグ → タイムラインの特定時間に配置

## 完了/渡しフィードバック

### 原則

- トースト通知は**使わない**
- うるさくない。すぐ消える。連続操作で積み重ならない
- ただし「ちゃんと聞いた」が伝わる

### 単発操作

- カードが ✓ アニメーションで縮んで消える（0.3s）
- それだけ。テキスト通知は出ない
- 気持ちいい消え方自体が「聞いた」のフィードバック

### 連続操作（2件目以降）

- 画面下部（BottomNav の上）に薄いバーが静かに出現
- 内容: 「✓ 3件  元に戻す」
- 同じバーの数字だけが更新される。バーは増殖しない
- 3秒間操作なしで自動消滅
- undo は最後の1件のみ
- undo 実行時: カードが元の位置にアニメーションで復帰。`ステータス` / `ballHolder` を操作前の値に復元
- アプリ遷移やページ離脱で undo キューはクリアされる（Firestore には即書き込み済みなので、undo は楽観的UIの巻き戻し + 再書き込み）

### 渡す操作

- カードが右に滑って消えるアニメーション（0.3s）
- 完了と異なる動き（右方向へスライドアウト）で「どこかに行った」感を出す
- 連続バーは完了と共通（「✓ 2件 渡し 1件  元に戻す」のように表記）

## 空の状態

### タスクゼロの日

タイムラインエリアに:
- 「空き 8時間」の空き時間ブロック（業務時間全体）
- その下にあとで整理トレーからのチップ提案
- 何もなければ中央に軽いテキスト: 「何もない一日。新しく追加するか、あとで整理から持ってきましょう」

### あとで整理トレーが空

- ヘッダー: 「あとで整理  0件」
- 展開時に軽いテキスト: 「＋ボタンで何でも入れておけます」

## タスク詳細

- タイムラインのカードをタップ → ハーフモーダルで詳細表示
- 表示: タスク名、PJ名、担当者、時間、所要時間、メモ、ボール状態
- 編集: タスク名、所要時間、メモは inline 編集可能
- 「渡す」「完了」ボタンもモーダル内に配置
- モーダル外タップまたは下スワイプで閉じる

## クイック追加（FAB）

- 画面右下にFAB（+ボタン）、BottomNav の上、ボトムシートヘッダーの右（`bottom: 80px; right: 20px` 付近）
- タップ → 軽いシートが上がる
- 最低限: タイトル入力のみ
- オプション: 所要時間（15分/30分/1h/2h/4h のチップ選択）、いつ（あとで / 時間指定）
- Any Planner の入力画面を参考にするが、更にシンプルに
- 「保存」ボタンは目立つが小さい。入力後すぐ閉じて次を入れられる

## BottomNav

- 3タブ: 工程表 / 今日 / 設定
- 今日がデフォルト選択
- ボール専用タブは作らない（あとで整理トレーに統合済み）

## ビジュアルスタイル

### カラー

| 要素 | 色 |
|------|---|
| 背景 | 白 (#FFFFFF) |
| テキスト | 黒 (#000000) / グレー (#6B7280) |
| カード border | 薄グレー (#E5E7EB) |
| 現在時刻ライン | 黄色 (#FACC15) |
| 完了チェック | グレー枠、タップ後黒塗り |
| チップ | 薄グレー背景、黒文字 |
| 期限超過 | 赤テキストのみ（背景色は使わない） |

### タイポグラフィ

| 要素 | サイズ | ウェイト |

| 日付数字 | 40-48px | Bold |
| 年月 | 14px | Regular, グレー |

| タスク名 | 16px | Semi-bold |

- 画面左右パディング: 20px

- カード間: 8px
- カード角丸: 12-16px
- 全体的に余白を大きめに取る

## 削除する既存要素

- 「APDW Project Compass」ヘッダー
- サマリー3列（自分ボール / 相手待ち / 空き時間）
- 「今日の進行」セクションタイトル + 説明文
- 「ボールを見る」ボタン
- ヘッダーカードと進行カードの分離
- 保存時のトースト通知（「タスクを完了にしました（ローカル保存）」）

## データ要件

既存の Task モデルをそのまま使う。新規フィールド不要。

タイムライン表示に使うフィールド:
- `タスク名` — カードタイトル
- `予定開始日` / `期限` — 時間配置
- `工数見積(h)` — 所要時間表示、空き時間計算
- `ballHolder` — 渡す操作の対象判定
- `responseDeadline` — 催促日（今日なら今日のタイムラインに表示）
- `ステータス` — 完了判定

## 技術方針

- Compass Web（React + TypeScript + Tailwind）内のレスポンシブ実装
- 既存の `App.tsx` TasksPage を大幅リファクタ
- `SwipeBallCard` コンポーネントを再利用（右スワイプ=渡す操作）
- ボトムシートは新規コンポーネント（CSS transitions + touch event handling。既存の SwipeBallCard と同じパターン。外部ライブラリは追加しない）
- 時間軸レンダリングは既存の `MOBILE_DAY_START_MINUTES` 等の定数を活用
- 現在の `App.tsx` TasksPage（3000行超）から Today ビューを独立コンポーネントに抽出
