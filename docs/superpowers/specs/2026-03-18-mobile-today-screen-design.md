# Compass モバイル「今日」画面 設計スペック

## 概要

Compass スマホ版の中核画面。目的は「タスク管理」ではなく「今日を前に進めること」。
Any Planner の時間軸 UI をベースに、Compass 独自のボール（受け渡し）概念を自然に統合する。

参考: https://any-planner.com/

## 設計原則

- 画面の主役は今日のタイムライン
- 情報量は最小限。管理ツールっぽさ、設定画面っぽさを消す
- 白/黒/グレー基調。アクセント色は現在時刻ラインの黄色のみ
- 余白大きめ、角丸大きめ、薄い境界線、影なし
- ボールは独立した表示概念ではなく、カードに対する操作の結果として存在する

## 概念モデル

### タイムラインに出るもの = 自分がアクションを取るもの

タイムラインには「自分が今日やること」だけが並ぶ。
相手待ちのタスクは出ない（自分がアクションを取れないため）。

例外: 催促日が今日の相手待ちタスクは出る（催促する = 自分のアクション）。

### カードに対する操作は2つだけ

| 操作 | ジェスチャー | 意味 | 結果 |
|------|-------------|------|------|
| 完了 | 右端の○タップ | やった | カード消える。done。 |
| 渡す | 右スワイプ | 自分のパートは終わり、次は相手 | 今日から消え、あとで整理トレーに沈む |

- どのカードでも両方の操作ができる
- 「平面図を書く」→ ○タップで完了
- 「田中さんに図面確認する」→ 確認した → 右スワイプで渡す → 自分の今日から消える
- システムが決めるのではなく、ユーザーが文脈で判断する
- どちらも undo 可能（後述のフィードバック仕様参照）

## 画面構成

### レイアウト全体

画面は**1つの連続した流れ**。カード分割やセクション分割はしない。
日付 → 週 → タイムライン → ボトムシートが自然に続く。

```
┌──────────────────────────────┐
│                              │
│  18  3月                     │  ← 大きな日付（画面の主役）
│  2026          < 今日 >      │  ← 年、前日/翌日ナビ
│                              │
│  日  月  火  水  木  金  土   │
│  15  16  17 ⬤18 19  20  21  │  ← 週の日付列
│                              │
│  8:00                        │  ← 時刻ラベル（左、小さくグレー）
│   ┌───────────────────┐  ○  │  ← タスクカード + 完了チェック
│   │ 施主打合せ資料作成   │     │
│   │ 8:00 - 9:00   1h  │     │
│   └───────────────────┘     │
│  9:00                        │
│                              │
│      空き 2時間              │  ← 空き時間ブロック
│      [見積確認] [メール返信]  │  ← チップ提案
│                              │
│  11:00                       │
│   ┌───────────────────┐  ○  │
│   │ 田中さんに図面確認   │     │
│   │ 11:00 - 11:30  30m│     │
│   └───────────────────┘     │
│                              │
│  ═══ 12:21 ═══════════════   │  ← 現在時刻ライン（黄色）
│                              │
│      空き 5時間30分          │
│      [現場写真整理] [書類提出]│
│                              │
├──────────── ▔▔▔ ────────────┤  ← ドラッグハンドル
│  あとで整理          4件     │  ← ボトムシート（常にここに見える）
├──────────────────────────────┤
│  🏠 工程表   ≡ 今日   ⚙ 設定│  ← BottomNav（3タブ）
└──────────────────────────────┘
            (+)                   ← FAB
```

### ヘッダー / 日付エリア

- 「APDW Project Compass」などのアプリ名は**表示しない**
- 日付の数字が画面最大のテキスト（Any Planner の "28" と同様）
- 「今日」ボタン（pill型）、<> で前日/翌日
- 週の日付列: 日〜土、当日は黒丸塗りつぶし
- 日付エリアとタイムラインの間にカード境界やセクションタイトルを**入れない**

### タイムライン

Any Planner 式の連続時間軸。

- 左側に時刻ラベル（8:00, 9:00, ... 小さくグレー）
- タスクカードがその時刻位置に配置される
- タスクがない時間帯は圧縮（完全等間隔ではない）
  - ただし空き時間ブロックは明示的に表示: 「空き 2時間」
  - 空き時間ブロック内に、あとで整理トレーから「ここに入りそうなもの」をチップで提案
  - チップタップ → その時間枠に配置
- 現在時刻: 黄色の横断ライン + 時刻バッジ（Any Planner と同様）

### タスクカード

- 白背景、角丸大きめ（12-16px）、薄いグレーの border、影なし
- 内容: タスク名（太字）、時間範囲、所要時間
- 右端に完了チェック○（タップで完了）
- 右スワイプで「渡す」操作
- 情報は最小限。PJ名・担当者などは**出さない**（タップして詳細で見る）

### 空き時間ブロック

- 背景色なし、テキストのみで軽く表示
- 「空き 2時間」を中央にグレーテキスト
- その下にチップ（pill型ボタン）で、あとで整理の中から候補を提案
- チップは3-4個まで。多すぎると騒がしい

### 現在時刻ライン

- 黄色の水平線がタイムライン全幅を横断
- 左端に現在時刻バッジ（黄色背景、黒文字）
- Any Planner の表現をそのまま踏襲

## あとで整理ボトムシート

### 基本動作

- **常に画面下部に1行分が見えている**（「あとで整理 4件」のヘッダー行）
- ドラッグハンドル（小さい横棒）で上スワイプして展開
- 展開するとタイムラインの上にオーバーレイ
- BottomNav の上に位置する

### 中身

**フラットなリスト。セクション分けなし。**

- 未整理のタスク（タイトルだけで雑に入れたもの）
- 渡して待っているタスク（相手待ち）
- 両者を同列に並べる

相手待ちのアイテムは、カード内にさりげなく「→ 田中  催促3/20」のように小さく表示するだけ。
それだけで「待ち」と区別できる。

```
├──────────── ▔▔▔ ────────────┤
│ あとで整理              4件  │
│                              │
│  ● 見積書の確認    30m    ○  │
│  ● 田中 → 図面FB  催促3/20  │
│  ● 現場写真整理           ○  │
│  ● 書類提出        15m   ○  │
│                              │
├──────────────────────────────┤
```

### あとで整理からの操作

- タップ → 詳細/編集
- ○タップ → 完了
- 右スワイプ → 渡す
- 長押しまたはドラッグ → タイムラインの特定時間に配置

## 完了/渡しフィードバック

### 原則

- トースト通知は**使わない**
- うるさくない。すぐ消える。連続操作で積み重ならない
- ただし「ちゃんと聞いた」が伝わる

### 単発操作

- カードが ✓ アニメーションで縮んで消える（0.3s）
- それだけ。テキスト通知は出ない
- 気持ちいい消え方自体が「聞いた」のフィードバック

### 連続操作（2件目以降）

- 画面下部（BottomNav の上）に薄いバーが静かに出現
- 内容: 「✓ 3件  元に戻す」
- 同じバーの数字だけが更新される。バーは増殖しない
- 3秒間操作なしで自動消滅
- undo は最後の1件のみ

### 渡す操作

- カードが右に滑って消えるアニメーション（0.3s）
- 完了と異なる動き（右方向へスライドアウト）で「どこかに行った」感を出す
- 連続バーは完了と共通（「✓ 2件 渡し 1件  元に戻す」のように表記）

## クイック追加（FAB）

- 画面右下にFAB（+ボタン）
- タップ → 軽いシートが上がる
- 最低限: タイトル入力のみ
- オプション: 所要時間（15分/30分/1h/2h/4h のチップ選択）、いつ（あとで / 時間指定）
- Any Planner の入力画面を参考にするが、更にシンプルに
- 「保存」ボタンは目立つが小さい。入力後すぐ閉じて次を入れられる

## BottomNav

- 3タブ: 工程表 / 今日 / 設定
- 今日がデフォルト選択
- ボール専用タブは作らない（あとで整理トレーに統合済み）

## ビジュアルスタイル

### カラー

| 要素 | 色 |
|------|---|
| 背景 | 白 (#FFFFFF) |
| テキスト | 黒 (#000000) / グレー (#6B7280) |
| カード border | 薄グレー (#E5E7EB) |
| 現在時刻ライン | 黄色 (#FACC15) |
| 完了チェック | グレー枠、タップ後黒塗り |
| チップ | 薄グレー背景、黒文字 |
| 期限超過 | 赤テキストのみ（背景色は使わない） |

### タイポグラフィ

| 要素 | サイズ | ウェイト |
|------|--------|---------|
| 日付数字 | 40-48px | Bold |
| 年月 | 14px | Regular, グレー |
| 時刻ラベル | 12px | Regular, グレー |
| タスク名 | 16px | Semi-bold |
| 時間範囲/所要時間 | 13px | Regular, グレー |
| 空き時間テキスト | 14px | Regular, グレー |
| あとで整理ヘッダー | 15px | Semi-bold |

### スペーシング

- 画面左右パディング: 20px
- カード内パディング: 16px
- カード間: 8px
- カード角丸: 12-16px
- 全体的に余白を大きめに取る

## 削除する既存要素

- 「APDW Project Compass」ヘッダー
- サマリー3列（自分ボール / 相手待ち / 空き時間）
- 「今日の進行」セクションタイトル + 説明文
- 「ボールを見る」ボタン
- ヘッダーカードと進行カードの分離
- 保存時のトースト通知（「タスクを完了にしました（ローカル保存）」）

## データ要件

既存の Task モデルをそのまま使う。新規フィールド不要。

タイムライン表示に使うフィールド:
- `タスク名` — カードタイトル
- `予定開始日` / `期限` — 時間配置
- `工数見積(h)` — 所要時間表示、空き時間計算
- `ballHolder` — 渡す操作の対象判定
- `responseDeadline` — 催促日（今日なら今日のタイムラインに表示）
- `ステータス` — 完了判定

## 技術方針

- Compass Web（React + TypeScript + Tailwind）内のレスポンシブ実装
- 既存の `App.tsx` TasksPage を大幅リファクタ
- `SwipeBallCard` コンポーネントを再利用（右スワイプ=渡す操作）
- ボトムシートは新規コンポーネント（react-spring or framer-motion）
- 時間軸レンダリングは既存の `MOBILE_DAY_START_MINUTES` 等の定数を活用

## 今回のスコープ外

- ボール専用ページ（将来フィルタで対応可能）
- Push通知
- オフライン対応
- カレンダー同期表示（Google Calendar イベント表示）
- ドラッグ&ドロップでの時間変更
