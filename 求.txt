1) 事業の狙いと商品の役割
1. Compass（主役）

目的
プロジェクトとタスクの管理を、建築・施工の現場/設計チームが「日々回せる状態」にする（工程/担当/締切/進捗/役割/招待を一元化） 

README

売り方
ツール（SaaS）として法人でも個人でも買える。料金は「1ユーザー=¥1,000/月」を基本にする。

2. AI×建築サークル（補助導線）

目的
学び・コミュニティ・イベント等の“人の成長”を提供する会員制。Compassの導入を邪魔しない位置づけにする（サークルはサークル、ツールはツール）。

関係性
サークル特典としてCompass利用を「組織あたり3席まで含む」に統一し、4席目以降はCompassの追加席（¥1,000/席）で増やせる形にする。

2) 体験設計として欲しい「挙動」
1. デモとトライアルの2段階

A. デモ（操作確認）
目的
誰でも気軽に触れる。保存されない仕様でOK（リロードで消える＝むしろ安心して試せる）。

実現の状態
デモモードが存在し、デモとして動かせる（環境変数 VITE_DEMO_MODE=true がある） 

.env

B. 14日トライアル（実運用）
目的
自分の組織・現場で「実データで回す」。ここが購入判断の本番。

挙動
トライアル開始→14日後に自動で課金へ移行（キャンセル可能）。法人でカードが難しいケースは、別導線（相談/請求書）で吸収できる余地を残す。

2. 料金と席数の考え方（最重要）

目的
料金は常に分かりやすく「1席¥1,000」。同時に、組織として人数が増える時にスムーズに課金が増える（売上が自然に伸びる）状態にする。

欲しい挙動

各組織には「契約席数（seatLimit）」がある

組織内のアクティブユーザー数が seatLimit を超えない

超えそうになった瞬間に、追加席購入の導線が出る（押し売りではなく“必要になった瞬間”）

前提として、課金の有効/無効は既にサブスク状態（active/trialing）で判定している 

org-setup

。これに「席数」を足すのがゴール。

3. サークル会員の席数ルール（5→3へ修正）

目的
サークル特典でCompassが「5席」になっている状態を「3席」に統一する。サークルが法人導入の抜け道にならず、かつ小チームが始めやすい特典として成立する状態にする。

欲しい挙動

サークル加入組織は seatLimit がベースで3席

4席目以降は「追加席（¥1,000/席）」で無制限に増やせる

サークル会員が複数いても “3席が増殖しない”（組織単位で3席固定）

3) Stripeまわりで欲しい「管理の形」

目的
席数の増減を、運用コストが最小で、請求事故が起きない形で回す。

欲しい挙動

組織オーナーが「人員管理」画面で現在の利用状況を見られる
表示例
契約席数（seatLimit）/ 利用中（activeユーザー数）

「＋席を追加」ボタンがある
押すと Stripe のカスタマーポータルへ遷移し、席数（数量）を増やせる

根拠
既にStripeポータルセッションを作るAPIがあり 

billing

、フロントにもポータル生成呼び出しがある 

api

。この既存導線を「席追加体験」の核にする。

支払い後に自動で反映される
Stripeのサブスク情報を同期し、組織の課金状態と席数が更新される（同期処理の骨格は既にある） 

billing

4) LPでやるべきこと（目的だけ）
1. Compass LP（主役）

目的
「プロジェクト・タスク管理ツールとしての価値」と「14日トライアル」への導線を最短化する。

LPが伝えるべきこと

Compassは何を解決するか（プロジェクト/タスク/工程の見える化と運用） 

README

デモは気軽に触れる（保存されない）

実運用は14日トライアル（自動課金へ）

価格は1席¥1,000でシンプル

人数が増える時の扱い（＋席で追加できる。上限は設けない）

サークルの載せ方（重要）
LPの主導線はCompass。サークルは「学びたい人向けの別商品」として補助リンクで置く。Compass購入の判断を邪魔しない。

2. サークル LP（補助）

目的
AI活用の学び・コミュニティを売る。ツールの話は「特典」として簡潔に。

載せるべきこと

サークルの価値（学び/テンプレ/イベント/壁打ち）

特典としてCompassが組織あたり3席まで含まれる

4席目以降は追加席で増やせる（会社利用へ自然にスケール）
